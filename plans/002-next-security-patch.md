## Why this matters

`npm audit` at the planned-at commit reports 6 vulnerabilities (4 high,
2 moderate). The high-severity ones chain through `next@16.1.6` and dev-only
tooling. npm's resolver says the fix for next is `16.2.9` and is NOT a semver
major (`fixAvailable: { name: 'next', version: '16.2.9', isSemVerMajor: false }`).
This is a production-deployed site (noobwork.no on Vercel), so the framework
patch is the one that matters. Separately, `package-lock.json` still says
`"version": "0.2.0.0"` while `package.json` says `"0.3.0.0"` — metadata drift
from a version bump that skipped `npm install`; the install in this plan
resyncs it.

## Steps

### Step 1: Upgrade next (and matching eslint-config-next)

```bash
npm install next@16.2.9 eslint-config-next@16.2.9
```

(`eslint-config-next` is pinned `16.1.6` in devDependencies and tracks the
next version; keep them in lockstep. If `16.2.9` of the config does not
exist, use the closest available `16.2.x` and note it.)

**Verify**: `node -e "console.log(require('./package.json').dependencies.next)"` → `16.2.9`

### Step 2: Confirm the lockfile resynced

**Verify**: `node -e "console.log(require('./package-lock.json').version)"` → `0.3.0.0`

### Step 3: Confirm the advisories are gone

**Verify**: `npm audit 2>&1 | tail -3` → next is absent from the report. Remaining
findings (if any) must all be in dev-only chains (eslint/vitest). Record the
new totals in your final report.

### Step 4: Full verification battery

**Verify**: `npm run lint && npx tsc --noEmit && npm run build && npm test` → all exit 0, build prerenders 13/13 pages.

### Step 5 (optional, only if Step 3 still shows fixable dev-only items):

```bash
npm audit fix
```

(No `--force`.) Re-run Step 4's battery afterwards. If `npm audit fix`
changes anything beyond dev-only transitive pins, revert it
(`git checkout -- package.json package-lock.json`, then redo Steps 1-4) and
note that in your report.
